## Frontend Output Display: Destination Cards

## Layout Overview

After the backend returns recommended destinations, the frontend displays them as interactive cards arranged in a scrollable grid layout.

Each card provides:

* Destination information
* Weather summary
* Tags and terrain
* Best time to visit
* Reason for recommendation
* Image preview
* Action button

---

## Card Elements

| Element                   | Example                                            | Description / Function                           |
| ------------------------- | -------------------------------------------------- | ------------------------------------------------ |
| Destination Name          | Goa, India                                         | Displayed prominently at the top.                |
| Weather Summary           | Sunny, 28°C                                        | Shows real-time or forecasted weather.           |
| Terrain & Tags            | Beach · Relaxing · Scuba Diving                    | Terrain type + mood/activity tags.               |
| Best Time to Visit        | November to January                                | Seasonal suitability.                            |
| Reason for Recommendation | “Matches your mood, climate, and activity filters” | Explains why this destination was selected.      |
| Image                     | Background photo                                   | High-quality image representing the destination. |
| Action Button             | View Details / Plan Trip                           | Opens the detailed destination page.             |
